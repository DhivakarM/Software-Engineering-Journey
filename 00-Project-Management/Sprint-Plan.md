# Sprint Success Criteria

Each sprint is considered successful when:

- Sprint objectives are completed.
- Planned deliverables are achieved.
- Documentation is updated.
- Git commits are completed.
- Changes are pushed to GitHub.
- Lessons Learned are recorded.

---

# Sprint Workflow

Every sprint follows the same engineering process:

Planning

↓

Learning

↓

Implementation

↓

Review

↓

Documentation

↓

Git Commit

↓

Git Push

↓

Sprint Retrospective

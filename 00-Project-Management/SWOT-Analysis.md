## 1. Purpose

This SWOT analysis evaluates the internal strengths and weaknesses, as well as the external opportunities and threats, associated with the Software Engineering Journey project.

The purpose is to support strategic decision-making, identify potential risks, and ensure that the project remains focused on its objective of developing practical software engineering capability.

---

# 2. Strengths

Strengths are internal advantages that support the success of the project.

### 2.1 Strong IT Experience

The project owner has approximately 22 years of previous IT experience, including experience as a Technical Project Manager and experience working with cross-functional technical teams.

### 2.2 Project Management Experience

Experience in project management and PMP certification provide strong capabilities in:

- Planning
- Goal setting
- Risk management
- Documentation
- Progress tracking
- Continuous improvement

### 2.3 Real-World Project Focus

The learning journey is connected to a real-world application:

**FinAdvisor-Pro — Personal Financial Advisor Application**

This provides a practical objective for applying the knowledge gained.

### 2.4 Structured Learning Approach

The project follows a structured learning and engineering process:

> Understand → Practice → Apply → Document → Review → Commit → Push

### 2.5 Professional Documentation

The project maintains structured documentation including:

- Project Charter
- Vision and Mission
- OKRs
- KPIs
- Project Dashboard
- Weekly Tracker
- Sprint Plan
- Risk Register
- Decision Log
- Lessons Learned

# 3. Weaknesses

Weaknesses are internal challenges that may affect project progress.

### 3.1 Limited Recent Hands-On Coding

Although the project owner has significant IT experience, current hands-on programming skills are being rebuilt through structured learning.

**Mitigation:**

> Learn → Practice → Build Small Applications → Apply to FinAdvisor-Pro

### 3.2 Broad Learning Scope

The project covers multiple areas, including:

- Python
- Databases
- REST APIs
- System Design
- AI
- LLM Fundamentals

**Mitigation:**

Follow the approved learning roadmap and focus on one major topic at a time.

### 3.3 Limited Weekly Time

The project is currently planned around:

- Two learning sessions per week
- Approximately one hour per session

**Mitigation:**

Focus on consistency and measurable progress rather than maximum study hours.

### 3.4 Initial Dependency on Guidance

At the beginning of the journey, significant guidance may be required to understand new technical concepts.

**Mitigation:**

Progress gradually from:

> Guided Learning → Collaborative Learning → Independent Engineering

### 3.5 Potentially Large Application Scope

FinAdvisor-Pro may become complex as additional features are identified.

**Mitigation:**

Begin with a Minimum Viable Product (MVP) and expand the application incrementally.

# 5. Threats

Threats are external factors that may negatively affect the project.

### 5.1 Loss of Momentum

Long-term learning projects may lose momentum due to competing responsibilities or inconsistent participation.

**Mitigation:**

Use:

- Weekly Tracker
- Sprint Plan
- Project Dashboard
- Git Commit History

### 5.2 Rapid Technology Changes

Programming languages, frameworks, AI tools, and software technologies continue to evolve.

**Mitigation:**

Focus first on fundamental and long-lasting software engineering concepts before learning rapidly changing tools and frameworks.

### 5.3 Scope Creep

New technologies and ideas may continuously be added to the learning roadmap.

**Mitigation:**

Record new ideas in the Decision Log or future backlog instead of immediately changing the current project scope.

### 5.4 Over-Engineering

There is a risk of spending excessive time planning, documenting, or designing instead of building and practicing.

**Mitigation:**

Maintain a healthy balance between:

> Plan → Learn → Build → Review

The project should increasingly focus on hands-on development as the learning journey progresses.

### 5.5 Increasing Technical Complexity

Advanced topics such as databases, system design, AI, and LLMs may introduce significant technical challenges.

**Mitigation:**

Break complex topics into smaller learning milestones and build knowledge incrementally.
